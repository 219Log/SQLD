2-2.SQL 활용

  • 순수관계 연산자
    • SELECT , PROJECT, JOIN, DIVIDE
  • 쿼리해석
    • NOT EXIST절의 연관서브쿼리에서 X.고객ID = B.고객ID로 되어있어 하나만 비선호 컨텐츠만들어가도 모든 컨텐츠가 배제됨
      • X.컨텐츠ID = B컨텐츠ID가 존재하지않아서
    • 조건문을 더 확인해보자
  • 쿼리사용
    • WHERE 절에도 조인조건 사용가능
  • FULL OUTER JOIN
    • LEFT JOIN 과 RIGHT조인을 UINON으로하여 같은값이 가능
    • INNER JOIN 조인후 A UNION ALL, B UNION ALL 조건에서 불포함한다 겹치는 부분을 하여 값은값가능
    • A, B 의 전체의 값을 나타냄
  • OUTER JOIN 문장
    • 왼 3 / 풀 5 / 오 4
  • LEFT JOIN
    • A LEFT JOIN B 의 경우 B가 값 이 없을경우 NULL 값으로 표현 되자나!!
  • 집합 연산자의 종류
    • UNION 합집합
    • UNION ALL 합집합에 중복행그대로
    • INTERSECT 교집합
    • EXCEPT 차집합 MINUS 와 같음

  • EXCEPT
    • 해당 문제는 아래와 같이 풀이할 수 있다.

  • EXCEPT는 NOT EXISTS()와 NOT IN으로 데이터 처리가 가능하다
    • 하지만 한번에 이어져야하는데 NOT IN 같은 경우 두개로 나뉘어서 다른 결과가 나옴
    • 붙이 면같은값일텐데
  • INTERSECT
    • 교집합이다 A,B 의 겹치는 부분 만 가져가려면 마이너스를 이용한다.
      • NOT IN (A MINUS B)
      • NOT EXISTS(A MIUS B)
      • IS NOT NULL 로 뺌
  • 데이터 모델 설명
    • 1대1 관계에서 EXCEPT 는 공집합
    • 1대1 관계에서 UNION은 전체건수 2배
    • 1대1 관계에서 INTERCEPT는 JOIN 과 UINON 과 같다
  • CONNECT BY
    • WHERE 절에 작성된 조건절과 다름
    • START WITH 절에서 필터링된 시작데이터는 결과목록에 포함됨
    • CONNECT BY PRIOR에 의해 필터링됨
      • 널 로우부터 시작하여 비트윈 안에있는 값만 나오게함
  • PRIOR
    • CONNECT BY 절에 자주 사용되며 PRIOR에 지정된 컬럼이 맞은편 컬럼을 찾아감
      • WHERE 과 SELECT 절에도 사용됨
      • PRIOR 자식 = 부모 순방향 \\\\ PRIOR부모 = 자식 역방향
  • 계층형 질의문
    • SQLServer에서의 계층형 질의문은 CTE를 재귀호출함으로 계층구조 전개
    • SQLServer에서의 계층형 질의문은 앵커 맴버를 실행하여 기본 결과 집합을 만들고 재귀 맴버를 지속적 실행한다.
    • 계층형데이터란
      • 동일 테이블에 계층적으로 상위와 하위데이터가 포함된 데이터를 말한다.
      • 예를들어 사원테이블에서 사원들사이에 사원 관리자 하위 관리자 단계적데이터
  • 계층형
    • 역방향으로 조인하여 데이터를 가져오고 PRIOR 로 범위잡아서
  • 메인쿼리와 서브쿼리
    • 서브쿼리는 단일행 또는 복수행 비교연산자와 같이 사용가능
    • 복수행 결과 서브쿼리는 IN, ALL, ANY등 복수행 비교연산자와 사용
    • 서브쿼리는 SELECT, FROM, HAVING, ORDER BY 절에서 사용가능하다
    • 다중컬럼 서브쿼리는 SQLServer에서 사용 불가능
  • NOT EXISTS()
    • 괄호안에 있는것을 빼고 출력한다.
    • 재외된 항목에 대한 내역은 3가지를 통해 표현가능
  • GROUP BY
    • 사용하지 않고 HAVING 을 사용했다고 문장에 오류가 발생하지 않는다.
  • 뷰
    • 독립성
      • 데이블 구조가 변경되어도 사용 응용프로그램 변형하지 않아도 됨
    • 편리성
      • 복잡한 질의를 뷰로 생성함으로써 관련 질의를 단순하게 작성할 수 있다.
      • 해당 SQL문을자주 사용할 때 뷰를 이용하면 편리하게 사용할 수 있다.
    • 보안성
      • 직원의 급여정보와 같이 숨기고 싶은 정보가 존재한다면 뷰에서 컬럼을 뺀다
    • 뷰는 단지 정의만 가지고 있어서 실생시점에서 질의를 재작성하여 수행함
    • 보안으로사용함
    • 뷰 기능 생성 DBMS 존재
  • GROUP BY ROLLUP
    • 각 그룹별 토탈하여 모으고 집계함수를 사용해야한다 (마지막 카운트 함수라던지)
    • CASE 절 GROUPING()
      • GROUPING
        • 해당 컬럼의 Row가 GROUP BY에 의해서 산출된 Row인 경우에는 0
        • ROLLUP이나 CUBE에 의해서 산출된 Row인 경우에는 1을 반환
  • grouping sets
    • 집계함수를 보다 편하게
    • 같은말임

    그룹핑에 괄호가 더있음 그루핑

  • 윈도우 함수
    • partition과 group by 구문은 유사함
    • partition 구문이 없으면 전체 집합을 하나로 partition으로 정의한것과 동일하다.
    • 윈도우함수 처리로 인해 결과 건수는 줄지 않는다.
    • 윈도우 함수 적용 범위는 partition 을 넘을수 없다.
    • partition by a order by b desc
      • 파티션별 비 에서 가장 높은걸 뽑는다
      • 이순신 중에서 90 과 80이 있으면 90을 출력 ㅇㅋ
    • 윈도우 함수 order by 에 집계함수 쌉가능
  • DCL
    • GRANT
      • 권한 부여
    • REVOKE
      • 권한 회수
    • 예시
  • ROLE
    • 많은 DBMS 사용자에게 개별적을 많은 권한을 부여하는 번거로움과 어려움를 해ㅅ하기 위한 다양한 권한을 하나의 그룹으로 묶어 놓은 논리적인 권한그룹
  • EXEUTE IMMEDIATE
    • DDL이나 PL/SQL이 문장을 실행할 때 사용
  • 저장 모듈(Stored Module)구현
    • 3가지를 구현 가능하다
    • 저장형 프로시저는 sql 로직과 함께 데이터베이스내에 저장해 놓은 명령문의 집합
    • 저장형 함수(사용자 정의함수)는 단독적으로 실행되기보다는 sql 문을 통하여 호출되고 그결과를 리턴하는 sql의 보조적인 역할
    • 트리거는 특정한 테이블에 인업데 문이 dml문이 수행되었을 때 자동으로 동작

    • 트리거 : 데이터 무결성과 일관성을 위해서 사용자 함수를 사용한다.
    • 트리거는 TCL 사용불가능
