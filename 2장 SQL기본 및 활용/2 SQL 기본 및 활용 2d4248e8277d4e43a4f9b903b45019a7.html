2.SQL 기본 및 활용

  • SQL 명령어 종류
  • PK 제약조건
    • PK를 지정하는 ALTER TABLE 문장에 문법 오류가 존재(수정식으로 기본키등록)
      • ALTER TABLE PRODUCT ADD PRIMARY KEY PRODUCT_PK ON (PROD_ID);
      • ALTER TABLE PRODUCT ADD CONSTRAINT PRODUCT_PK PRIMARY KEY (PROD_ID);
      • 알터를 쓸때는 위와같이
    • NOT NULL은 제약 조건에 쓰지않는다.
    • 테이블을 생성할 때 PK를 지정하는 문장 오류 ADD 알터테이블 없이 들어가면안됨
    • 테이블 생성할 때 기본키등록
      • (… COSTRAINT PRODUCT_PK PRIMARY KEY(PROD_ID) );
  • SQL Server 테이블 수정 시 올바른 문장
    • ALTER TABLE 기관분류 ALTER COLUMN 분류명 VARCHER2 (30); ALTER TABLE 기관분류 ALTER COLUMN 등록일자 DATE NOT NULL;
    • SQLServer에서는 여러개의 칼럼을 동시에 수행하는 구문은 존재하지 않음
    • SQLServer는 괄호를 사용하지 않음
    • 제약조건을 NOT NULL을 입력하지않으면 NULL로 자동 등록
  • 테이블 컬럼 제약조건
    • 기본키(PRIMARY KEY)
      • 테이블당 하나만 생성가능하다
        • 하나만이라는게 컬럼이 아니라 구문을 한번실행가능
      • 컬럼여러개를 합친 복합키가 기본키가 될 수 있다.
    • 유니크(UNIQUE)
      • 중복값을 허용하지 않음
      • NULL값 가능
      • 테이블에 여러개 생성가능
      • 기본키 OR 유니크 하나만 사용가능
  • 테이블 구조 정의
    • 기본키 설정
      • 컬럼 데이터타입 [PRIMARY KEY],
      • CONSTRAINT 테이블_PK PRIVARY KEY(컬럼);
      • ALTER TABLE 테이블명 ADD CONTRAINT 테이블명_PK PRIMARY KEY (컬럼);
    • 인덱스 생성
      • CREATE INDEX 인덱스이름 ON 테이블이름 (컬럼);
      • ALTER TABLE 테이블 이름 ADD INDEX 인덱스이름 (컬럼);
      • 특징
        • 기본키는 자동적으로 인덱스를 생성
        • 빠르게 찾기가능
        • 무결성

      CREATE EMP

      (

      EMP_NO VARCHER2(10) PRIMARY KEY,

      EMP_NM VARCHER2(30) NOT NULL,

      DEPT_CODE VARCHAR2(4) DEFAULT ‘0000’ NOT NULL,

      JOIN_DATE DATE NOT NULL,

      REGIST_DATE DATE

      );

      CREATE INDEX IDX_EMP_1 ON EMP (JOIN_DETE);

  • 테이블 제약조건
    • Check
      • 무결성을 유지하기위한 컬럼에 설정하는 제약
    • 기본키(primary key)
      • 하나만 가능
    • 고유키(unique key)
      • null 가능
      • 여러개가능
    • 외래키
      • 다른테이블 참조
  • 칼럼 삭제 SQL문
    • ALTER TABLE EMP DROP COLUMN COMM;
    • 컬럼을 삭제하는것
    • 테이블삭제는 DROP TABLE 테이블명;
  • 테이블 이름 변경 SQL 문
    • RENAME STADIUM TO STADIUM_JSC;
  • 참조 관계 정의
    • 위는 부모테이블 아래는 자식테이블 적용
      • 마스터가 부모 자식이 자식
  • SQL INSERT 문
    • INSERT INTO 테이블 (컬럼) VALUES(값);
    • INSERT INTO 테이블 VALUES (*값);
      • 1번 : 컬럼 없으면 모든값 내야함
      • 2번 : 데이터 타입이잘못
      • 3번 : NOT NULL인데 생략함
  • 삭제문 비교
  • 격리성이 낮을 경우 발생할 문제점
    • Dirty Read
      • 다른 트랜잭션에 수정되었지만 아직 커밋되지 않은 데이터 읽은것
    • Non-Repeatable Read
      • 한 트랜잭션에 두번수행되어 두 쿼리가 잘못 나오는 현상
    • Phantom Read
      • 한 트랜잭션에 두번 실행되었을 때 첫번째 없던 유령 레코드가 두번째 나오는 현상
    • 원자성은 목표지 문제점이 아님
  • 널값 조회는 무조건 is not null 만사용가능
  • SQL 문제
    • 오라클에서 ‘’값으로 입력하면 null이 됨
    • 오라클에서 널값을 조회하려면 is null 해야함
    • sqlServer 의 경우 ‘’ 공백이며 조회하려면 =’’ 로 해야함
  • 단일행 문자형 함수 종류
    • 아스키 값 CHR(10) → 줄바꿈이다
  • SQL CASE 문

    SELECT LOC,

    CASE WHEN LOC = ‘NEW YORK’ THEN ‘EAST’

    ELSE ‘ETC’

    END AS AREA

    FROM DEPT;

    SELECT LOC,

    CASE LOC WHEN ‘NEW YORK’ THEN ‘EAST’

    ELSE ‘ETC’

    END AS AREA

    FROM DEPT;

    • 위아래 같은 내역임
  • 45번 틀린거 널 값 찿을때 IS NULL 또까먹음 하
    • NVL = IS NULL 같은거다
  • COALESCE(콜랩스)
    • 함수 사용시 아래 값을 따지면
      • 1행 : 1 + 2행 : 2 + 3행 : 3 = 정답은 6
  • 나누기
    • 분모에 0이 들어가면 에러뜸
    • 숫자와 계산에 널이 들어가면 무조건 널
    • 0/300 5000/0 1000/NULL
  • SELECT 계산식
    • AVG 등과 같은 계산식에 로우가 NULL 이 들어가면 제외
    • WHERE 절로 NULL값이 채택되어도 계산할때는 제외
  • 건수 나열
    • NULL 값은 카운트에 제외
    • ‘널’ ≠ NULL 저건문자임
  • 중첩된 집계함수는 1건이 될수 밖에 없어서 그룹바이를 써도 안된다.
  • 실행결과
    • 999 값은 0으로 치환하여 계산
    • 건수가 2개이며 제일 낮은 값 999가 앞에 100이 뒤에
  • ORDER BY
    • AS 명이나 컬럼 순서를 나타내는 정수도 사용 가능 하지만 혼용은 불가능
